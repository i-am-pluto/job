# GitHub Profile Design

## Goal

Build a GitHub profile README for `github.com/i-am-pluto/i-am-pluto` that feels personal and creative while still showing credible backend, data, AI-tooling, and indie-builder experience.

## Direction

Use a blended **cosmic hacker + indie builder** voice. The profile should feel like `i-am-pluto` first, with light space/system language, but the content should remain concrete and professional.

## Profile Shape

The README opens with a compact identity:

- `hi, i'm pluto`
- backend systems engineer
- AI-tooling tinkerer
- indie builder of small useful experiments

The main sections are:

1. **Currently Orbiting**: current focus areas such as backend platforms, AI agents, data systems, and shipped side projects.
2. **Things I Build**: grouped project types instead of an exhaustive repo list.
3. **Featured Transmissions**: curated links to selected repos and live projects.
4. **Stack Constellation**: practical skills grouped by domain.
5. **Telemetry**: optional GitHub stats cards or lightweight badges.
6. **Open Channels**: contact links.

## Content Choices

Highlight selected public repos and local resume truth:

- `Ai_Portfolio_AN` for AI/portfolio experimentation.
- `HabitForge` for a shipped productivity app.
- `solid-couscous` and `paras-website` for web experiments with live deployments.
- `indoor-positioning-system-for-visually-impaired` for ML/computer vision and assistive-tech work.
- `discordBot` for early automation.
- Current professional work in Java/Kotlin backend systems, Spark, Kubernetes, AWS, observability, MCP, and LLM agents.

Do not list every public repo. Older assignment, fork, and low-signal repos remain discoverable through GitHub but should not dominate the profile.

## Style Constraints

- Text-first Markdown with tasteful badges.
- No giant icon wall.
- No excessive animated widgets.
- No fake enterprise portfolio tone.
- Keep the language playful but not unclear.
- Use links that work from GitHub.

## Target Artifact

Create a local profile repo/artifact at `github-profile/i-am-pluto/README.md`. This can later be pushed to the special GitHub profile repo `i-am-pluto/i-am-pluto`.
